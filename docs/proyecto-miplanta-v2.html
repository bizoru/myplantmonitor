01 · ¿Qué es Miplanta?

Miplanta es un sistema completo de Internet de las Cosas (IoT) que monitorea una planta real en tiempo real. Los sensores miden temperatura, humedad del aire y humedad del suelo. Esos datos viajan por internet hasta una pantalla visible desde cualquier navegador.

El sistema también permite controlar la luz de crecimiento LED de la planta desde cualquier lugar del mundo con un clic.

ANALOGÍA — Es como un termostato inteligente, pero para una planta. La planta "habla" con la nube cada 60 segundos y la nube guarda todo el historial.

06 · Guía de Demostración — 3 Minutos

CONSEJO — Practica esta secuencia al menos dos veces antes de la presentación. Abre las 3 pestañas con anticipación para no perder tiempo buscando URLs.

Parte 1 · Los Datos en Vivo (1 minuto)

  1. 1
    Abre https://miplanta.app/ en el navegador
  2. 2
    Señala las gráficas de temperatura, humedad y humedad del suelo
  3. 3
    "Estos datos vienen del sensor conectado a la planta real, cada 60 segundos. InfluxDB los guarda durante 30 días para ver el historial."

Parte 2 · El Simulador 3D (1 minuto)

  1. 1
    Abre https://miplanta.app/sim/
  2. 2
    Mueve los sliders de temperatura o humedad → los valores cambian en la pantalla 3D en tiempo real
  3. 3
    Presiona el botón PUBLICAR
  4. 4
    "El dato acaba de viajar desde este navegador, pasó por internet, llegó al servidor en Virginia (AWS), y en segundos aparece en Grafana."
  5. 5
    Vuelve a Grafana y muestra el nuevo punto en la gráfica

Parte 3 · Controlar la Luz (1 minuto)

  1. 1
    En el simulador, activa el interruptor de LUZ
  2. 2
    La tira LED de la planta 3D se enciende con luz rosada/morada — igual que los LEDs de cultivo reales
  3. 3
    "Este comando viajó del navegador al servidor, y de ahí al microcontrolador ESP32 físicamente conectado a la planta. Todo en menos de 1 segundo."

07 · Preguntas Frecuentes de la Profesora

PreguntaRespuesta sugerida
¿Qué es IoT? Internet de las Cosas: objetos físicos conectados a internet que envían datos. Igual que un termostato inteligente o un reloj deportivo.
¿Por qué MQTT y no HTTP? MQTT usa muy poca batería y ancho de banda — perfecto para sensores pequeños que envían datos frecuentes. HTTP está pensado para páginas web.
¿Dónde están los datos? En un servidor AWS en Virginia, EE.UU. Base de datos InfluxDB, especializada en datos que cambian con el tiempo.
¿Qué aprendiste tú? "Aprendí a usar Node-RED para crear flujos de datos. Entendí cómo los sensores se comunican por internet y cómo visualizar datos en tiempo real."
¿Cómo investigaste para este proyecto? "Investigué sobre IoT, MQTT y servidores en la nube. Armé los flujos en Node-RED y preparé la presentación."

08 · Glosario Rápido

IoT
Objetos físicos conectados a internet que envían y reciben datos
MQTT
Protocolo de mensajes para máquinas — como WhatsApp para sensores
Broker
Servidor que recibe y distribuye mensajes MQTT (Mosquitto)
Topic / Tema
El "canal" de un mensaje MQTT — ej. sensors/plant-01/state
Node-RED
Programa visual para conectar sensores con bases de datos
InfluxDB
Base de datos especializada en datos que cambian con el tiempo
Grafana
Programa para crear gráficas de datos en tiempo real
ESP32
Microcontrolador con WiFi que conecta sensores al internet
DHT22
Sensor de temperatura y humedad del aire
API
Interfaz para que programas se comuniquen entre sí por internet
HTTPS
Versión segura de HTTP — el candado en el navegador
Docker
Sistema que empaqueta programas para correr igual en cualquier servidor
AWS
Amazon Web Services — servidores en la nube de Amazon
Time-series DB
Base de datos para mediciones ordenadas por tiempo
WebSocket (WSS)
Conexión de internet bidireccional en tiempo real
Caddy
Servidor web que gestiona HTTPS automático
